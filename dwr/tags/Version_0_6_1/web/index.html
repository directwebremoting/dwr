DWR - Direct Web Remoting

DWR is AJAX and XMLHttpRequest made easy. It makes it simple to call Java on the server directly from Javascript in the browser marshalling all the parameters and return values.

It gets rid of almost all the boiler plate code between the web browser and your Java code.

DWR helps you in producing highly interactive web-sites by providing some Javascript libraries to help with DHTML and by giving you a set of examples to copy from.

DWR fits in with your current servlet framework, it does not take over. It comes with Spring beans integration allowing you to remote your spring beans to Javascript, and it is being used in the next version of Webwork.

DWR is open source, licensed under the ASL version 2.0, with full intention that it be GPL compatible.

Examples

We have 4 tutorials/demos of using DWR to create:

  • Selection Lists

Finding out more ...

CVS, Issuezilla and the mailing lists are hosted by Sun at Java.net which is also the best place for downloads.

To help get started there is an introduction on how to add the jar file to your webapp, and then details on how to configure your WEB-INF/web.xml and dwr.xml files. There is also a reference to the dwr.xml DTD.

There are 2 fairly static Javascript files in addition to the files that are generated dynamically from your Java classes. engine.js and util.js are documented.

Related Projects

Sarissa is a cross-browser Javascript XML library. It contains better versions of some of the XML utilities in util.js. Don't look to Sarissa for remoting, but do look to it for better XML functions. Sarissa is GPL/LGPL.

Both cross-browser.com and quirksmode.org contain useful cross-browser script libraries.

JSON-RPC-Java does a similar job to DWR. The main difference between JSON-RPC-Java and DWR is that JSON focuses on a standardized wire protocol so you can re-write your back end server and leave the browser as it is. DWR focuses on making remoting easy for both ends of the spectrum, Java and JavaScript.
